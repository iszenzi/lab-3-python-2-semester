# Лабораторная работа №3 - Очередь задач: итераторы и генераторы    

## Описание
Проект демонстрирует создание пользовательских коллекций, реализацию протокола итерации (итераторы), а также использование генераторов для ленивой обработки и фильтрации задач на примере очереди TaskQueue.

## Структура проекта
```
|-- src/
|   |-- __init__.py
|   |-- main.py           # CLI для запуска и демонстрации работы
|   |-- loader.py         # Загрузчик задач 
|   |-- sources.py        # Источники задач
|   |-- protocol.py       # typing.Protocol для источников
|   |-- queue.py          # Очередь задач (TaskQueue) с итераторами и генераторами

```
## Реализованные компоненты

### Очередь задач (TaskQueue)

- Пользовательская коллекция (реализует `typing.Iterable`), хранящая список задач.
- Поддерживает многократный обход и получение длины `__len__`.
- При вызове `__iter__` возвращает новый экземпляр пользовательского итератора.

### Пользовательский итератор

- `TaskQueueIterator` — класс, реализующий протокол итерации (наследует `typing.Iterator`).
- Хранит внутреннее состояние текущего обхода (индекс).
- Самостоятельно управляет выбросом исключения `StopIteration` по окончании коллекции.

### Ленивые фильтры (Генераторы)

- Использование `yield` для возврата подходящих задач «на лету» без загрузки всего отфильтрованного списка в память.
- Спроектированы независимые генераторы:
    - по статусу (`filter_by_status`)
    - по приоритету (`filter_by_priority`)
    - готовые к выполнению (`get_ready_tasks`)

### Интеграция с CLI

- Команда `process` (`python -m src.main process`), демонстрирующая работу очереди через запуск задач и изменение их статусов.
- Добавлены флаги `--status` и `--priority` для применения фильтрации через генераторы напрямую из терминала.

## Установка зависимостей 
```
pip install -r requirements.txt
```

## Запуск программы
Вывод общей информации для взаимодействия с программой:
```
python -m src.main --help
```
Подробнее про запуск программы (обработка задач):
```
python -m src.main process --help
```
Пример загрузки 5 сгенерированных задач в очередь и фильтрации результата по приоритету 3:
```
python -m src.main process --generator 5 --priority 3
```
Запуск тестов с отчетом о покрытии кода:
```
pytest --cov -v
```
