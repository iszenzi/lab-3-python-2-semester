# Лабораторная работа №2 - Модели задач: дескрипторы и @property    

## Описание
Проект демонстрирует применение принципов ООП, инкапсуляцию, защиту инвариантов бизнес-логики, использование свойств (@property) и дескрипторов (Data и Non-data) на примере модели задачи.

## Структура проекта
```
|-- src/
|   |-- __init__.py
|   |-- main.py           # CLI для запуска и демонстрации работы
|   |-- loader.py         # Загрузчик задач 
|   |-- sources.py        # Источники задач
|   |-- protocol.py       # typing.Protocol для источников
|   |-- task.py           # Модель задачи (управление статусами, инкапсуляция)
|   |-- descriptors.py    # Дескрипторы для валидации и значений по умолчанию
|   |-- exceptions.py     # Иерархия пользовательских исключений
|   |-- logger.py         # Логгер
|
|-- tests/
|   |-- __init__.py
|   |-- test_sources_loader.py # Тесты источников и загрузчика
|   |-- test_task.py           # Модульные тесты доменной логики задачи
|
|-- tasks.json            # Пример входных задач
|-- tasks_no_defaults.json # Пример неполного payload

```
## Реализованные компоненты

### Модель задачи

- Task — объект бизнес-логики со строгими инвариантами.
- Свойства (@property) — read-only доступ к id, created_at и status, а также вычисляемое свойство ready_for_execution.

### Дескрипторы

- PriorityDescriptor — Data-дескриптор для валидации приоритета (проверка типа int и границ от 1 до 5).
- DefaultDescriptionDescriptor — Non-data дескриптор, предоставляющий текст ("Без описания") по умолчанию.

### Пользовательские исключения

- TaskError — базовый класс доменных ошибок.
- InvalidPriorityError — ошибка при передаче или установке некорректного приоритета.
- InvalidStateError — попытка запрещенного перехода состояния (например, нельзя завершить неначатую задачу).

### Интеграция старых источников

- TaskSource и загрузчики (FileTaskSource, ApiTaskSource, GeneratorTaskSource) адаптированы для корректной распаковки параметров и передачи их в новую ООП-модель задачи.
